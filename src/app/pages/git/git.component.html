Git

Comandos básicos

  • Clonar un repo en mi pc git clone url
  • Detectar cambios con respecto al repo subido: git status
  • Agregar los cambios que se van a subir: git add (carpeta o archivo)
  • Agregar todos los cambios que se quieren commitear: git add .
  • Commitear los cambios agregados: git commit -m "mensaje"
  • Subir los cambios a la rama dev por ejemplo: git push origin dev
  • Reservar los cambios para aplicarlos después: git stash
  • Bajar todas las ramas de un repo: git pull origin
  • Bajar/actualizar una rama de un repo: git pull origin nombre-rama
  • Verificar la url del repo: git remote -v
  • Reemplazar el repositorio remoto vinculado: git remote set-url origin https://...
  • Cambiar de rama: git checkout nombre-rama
  • Crear rama desde la rama donde estoy: git checkout -b nombre-rama
  • ¿En qué rama estoy? git branch
  • Borrar una rama: git branch -d nombre-rama
  • Ver historial de commits git log

Cómo subir y publicar un proyecto Angular en GitHub (GH-Pages)

  • Crear proyecto angular localmente
  • Instalar de manera global: npm install -g angular-cli-ghpages
  • Crear repo en GitHub y vincular nuestro repo de manera local: git remote add origin https://github.com/USER/REPOSITORY.git
  • Subir el repo: git push -u origin master
  • Ejecutar: ng build --prod para que se arme correctamente la carpeta dist/proyecto
  • Empaquetado de la aplicación, ejecutar: ng build --prod --base-href https://USERNAME.github.io/REPOSITORY/ (importante barra final)
  • Ejecutar para publicar: npx ngh --dir=dist/proyecto (con poner la primer letra del nombre del proyecto y tabular alcanza)
  • Entrar al repo en github -> settings -> GitHub Pages -> Cambiar source a gh-pages branch
  • Sitio publicado en: https://USERNAME.github.io/REPOSITORY/

Cómo actualizar el repo:

  • Commitear cambios y subirlo a master
  • Ejecutar: ng build --prod
  • Ejecutar: ng build --prod --base-href https://USERNAME.github.io/REPOSITORY/
  • Ejecutar para publicar: npx ngh --dir=dist/proyecto
